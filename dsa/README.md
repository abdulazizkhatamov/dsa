## Rules for Learning DSA

1. **Do not skip days.** Each day builds on the previous one.
2. **Read slowly.** Don't rush to the code. Understand the idea first.
3. **Write the code yourself.** Don't copy-paste. Type it out.
4. **Solve the practice problems before looking at the answer.**
5. **If something is unclear — re-read it. Then re-read it again.**

## How each file is structured

Every day file has:
- **Simple explanation** — what is this thing, in plain English
- **Real-world analogy** — connect it to something you already know
- **How it works** — step by step, no skipping
- **JavaScript code** — with every line explained
- **Practice problems** — you solve these yourself
- **Solutions** — with full explanation
